# v0.63.0

The v0.63.0 release report. Theme: the Rich Prompt composer moves onto chan's main WYSIWYG editor, and a devserver whose control script dies keeps a readable terminal and reconnects on demand instead of vanishing. Reconstructed from the commit range `7ac84ca4..92d84dd3` (33 commits: the rc1 delivery, the adversarial review and gap-closure round, the host smoke-test refinements, and the GA bump) plus the round's `dev/v0.63.0/` coordination tree (`review-round-plan.md`, `review-report.md`, `stage2-assignments.md`, `gap-closure-plan.md`, `host-decisions.md`, `host-scope-additions.md`, the four `findings-*.md`, and the delivery-team journals). Cut GA `v0.63.0` on 2026-07-03.

## The round

The rc1 delivery was three commits (the rich-prompt-over-Wysiwyg move, control-session reaping, and the survey close flow), authored by less experienced agents. Before it was trusted at the project bar it went through an adversarial correctness review and a gap-closure round, planned in `dev/v0.63.0/review-round-plan.md`.

Five agents (a Lead plus two rustaceans and two webdevs) reviewed every commit under complementary lenses: server and protocol correctness, desktop lifecycle and concurrency, Svelte and CM6 runtime, and empirical end-to-end. A read-only stage-1 pass produced each reviewer's findings; a stage-2 pass verified every finding with someone other than its author; stage 3 consolidated them in `dev/v0.63.0/review-report.md`. The round's currency was empirical verdicts, not diff reading: the project's own record is that static gates miss Svelte runtime errors and wire-string breaks, so wire strings were grepped across the whole repo including `desktop/` and `gateway/`, and WebView-specific behavior was flagged for desktop hand-smoke rather than declared from Chrome alone.

The review confirmed six must-fix bugs plus a release-cut blocker (the branch base predated the released v0.62.0), and surfaced seven design questions where the delivery had reversed documented prior decisions. The host resolved those in `dev/v0.63.0/host-decisions.md`, and the gap-closure round (`dev/v0.63.0/gap-closure-plan.md`) landed each fix as its own commit carrying a regression test. The reported P0, a connected control window whose Focus and eye buttons were inert, resolved to a stale-binary skew rather than a live bug at HEAD; a latent mis-route in `resolve_label_from` was hardened regardless and confirmed by the host's fresh-build WKWebView smoke. The host then smoke-tested and refined the branch across eight further commits before the cut.

## What shipped

Grouped as the CHANGELOG `[v0.63.0]` section records them:

- **The Rich Prompt composer is the main editor.** Cmd+Shift+P composes in the same WYSIWYG editor as the rest of chan, so a prompt gets inline image rendering, list and markup editing, and the editor's keymap. A pasted image renders inline while you compose and is delivered to the agent as an absolute on-disk path, so the agent reads it regardless of its working directory. The delivery format was the round's sharpest design question (a markdown embed with a leading `!` breaks a plain shell and is cwd-dependent); the host chose the cwd-independent bare absolute path.
- **Reconnect a stuck devserver from its workspace window.** When a devserver's control connection drops, each of its workspace windows shows a reconnecting overlay with a Reconnect button beside Abandon on chan-desktop. Reconnect closes the dead control terminal and re-runs the connection, the same flow the launcher's Connect drives. This was a host scope addition during the round (`dev/v0.63.0/host-scope-additions.md`), replacing an informational subline the overlay no longer needs.
- **A dead control script leaves a readable terminal instead of a vanished connection.** When a devserver's control script exits, chan marks the connection down but keeps the control terminal open at "process exited" so you can read why it died; the devserver's launcher identity dot turns red and its control row keeps a slow-flashing eye, the launcher stops offering that devserver's workspace and window rows so a click cannot land on a dead connection, and the workspace windows show a reconnecting overlay. The devserver stays un-reconnectable until you close that control terminal, after which it is ready to connect again; reconnecting never happens on its own.
- **A survey resolved in one window clears it in the others.** Answering, cancelling, or timing out a survey closes it in the other windows of its tab group, and the answering window is excluded from the close fan-out so it never sees a spurious "answered elsewhere". An unrelated Rich Prompt composer open at the time is left untouched.
- **A prerelease tag no longer updates the GA self-upgrade pointer.** Publishing a prerelease (`-rc`) tag ships its build as GitHub Release assets but leaves `/dl/cli/latest.json` and the desktop-updater manifest on the current GA version, so a release candidate cannot auto-upgrade GA installs; only a GA tag moves the pointer.

Smaller refinements from the host smoke pass: the connection-lost cue is the red identity dot rather than a flashing text pill; a control script that dies mid-connect fails fast with its own reason instead of a misleading come-up timeout; an empty pane shows a dotted backdrop; splitting a pane has a direct web keyboard shortcut (Ctrl+Alt+/ right, Ctrl+Alt+? bottom); the launcher's Focus and show/hide resolve a control terminal's native window directly; and list Tab / Shift-Tab step between real indent columns instead of a blind two spaces that dropped a list into a dead band.
