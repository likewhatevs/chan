## Goal

Two related deliverables to make the v0.13.0 cut (the
public-flip release) shippable:

1. **`CHANGELOG.md`** at repo root with v0.12.0 + earlier
   tagged releases summarised in Keep-a-Changelog format.
2. **Workflow audit + release-pipeline final
   verification**: review `.github/workflows/*.yml` for
   dead steps, outdated patterns, redundant matrix entries,
   missing concurrency limits. Produce a short report at
   task tail.

Round-3 Track 3 (cleanup / release readiness) per

## Acceptance criteria

### CHANGELOG.md

* Keep-a-Changelog format
  (https://keepachangelog.com/en/1.1.0/).
* Entries per tag in reverse-chronological order:
  v0.12.0 → v0.11.2 → v0.11.1 → v0.11.0 → v0.10.x
  (skip if minor) → older minor cuts.
* v0.12.0 entry: cite the chan-v0.12.0 annotated-tag
  message verbatim or paraphrase tight (Drafts +
  Team features + Hybrid Nav + right-click menus +
  screensaver + cursor saga). The annotated message
  already exists; reuse its prose.
* Per-entry sections: `### Added` / `### Changed` /
  `### Fixed` / `### Removed` where relevant.
* `## [Unreleased]` section at top, empty.
* Entries link to GitHub compare URLs (e.g.
  `[v0.12.0]: https://github.com/fiorix/chan/compare/chan-v0.11.2...chan-v0.12.0`).
  These work once the repo flips public.

### Workflow audit

Review `.github/workflows/*.yml`:

* `ci.yml` — per-PR gate. Ubuntu + macOS matrix
  (Windows out per `ci-13`). Look for: dead `env:`
  entries, redundant `actions/checkout` steps,
  unpinned action versions (`actions/checkout@v4` is
  pinned; spot-check the rest).
* `release.yml` — chan CLI release on `chan-v*` /
  `v*` tags. Last fired green on chan-v0.12.0
  (run 26332086305). Verify the `if:` extension
  from `ci-14` + the VERSION-strip fix are in HEAD;
  flag any divergence.
* `release-desktop.yml` — chan-desktop release on
  `chan-v*` tags. Same green on chan-v0.12.0
  (run 26332086302). Verify the download-pattern +
  upload-files broadening from `ci-14` are in HEAD.
* Any other workflow files: read + verify
  intentionality.

Report at task tail: each workflow + findings + any
fix recommendations (no fixes in this task; cut a
follow-up if needed).

## How to start

1. `git tag --list 'chan-v*' 'v*' | sort -V` → enumerate
   tags to summarise.
2. `git log --oneline <prev-tag>..<tag>` per pair to
   extract changes; group by Added/Changed/Fixed.
3. Read the chan-v0.12.0 annotated-tag message for the
   v0.12.0 entry seed.
4. Draft CHANGELOG.md.
5. Workflow audit: read each `.github/workflows/*.yml`;
   note findings.
6. Report appended to this task file tail.
