# ci-12 — workspace-wide GTK deps in CI test jobs (unblock ci.yml + release.yml)

Owner: @@CI
Cut: 2026-05-21 by @@Architect
Status: dispatched

## Goal

Restore the per-PR CI gate. Install the GTK development
libraries in every workflow job that runs `cargo clippy
--all-targets` / `cargo test` from the workspace root so
`desktop/src-tauri`'s transitive `glib-sys` dep compiles
under CI.

`ci.yml` has been **red since ~2026-05-19** (~15 consecutive
runs failing on the same `glib-sys` missing-pkg-config error
on `ubuntu-latest`). The release.yml smoke dispatch (run
26227752597) hit the same root cause. The gate was de facto
non-load-bearing for that window — local pre-push hooks were
the actual gating mechanism.

## Background

Surfaced 2026-05-21 by @@CI during the `ci-11` workflow_dispatch
smoke validation. See
[`../alex/event-ci-architect.md`](../alex/event-ci-architect.md)
"LATENT GAP: this is workspace-wide, NOT a release.yml issue".

Root cause: `desktop/src-tauri` is a workspace member
(`Cargo.toml [workspace.members]`); `cargo clippy
--all-targets` from the workspace root walks it, pulling
`glib-sys` → needs `libglib2.0-dev` on Ubuntu (and the GTK
stack at large for Tauri 2). Only `release-desktop.yml`
installs the GTK stack (`libwebkit2gtk-4.1-dev`,
`libayatana-appindicator3-dev`, `librsvg2-dev`,
`libsoup-3.0-dev`, `patchelf` at lines 114-123) — and its
build job runs `make build` not workspace clippy, so the
gap stayed latent until clippy ran in the smoke dispatch.

Local pre-push hooks pass because the developer
workstation has GTK installed (chan-desktop devs need it
for `make run`). CI runners are fresh — they don't.

## Decision: fix shape (a) — apt-install GTK deps

@@Architect routed option (a) per @@Alex 2026-05-21
("install gtk dev as you recommend"). Heavier than the
exclude-from-workspace alternative but preserves coverage:
the chan-desktop crate continues to get per-PR clippy +
test on Ubuntu. Matches `release-desktop.yml`'s existing
GTK install shape so the patterns stay symmetric.

The exclude-from-workspace alternative (option (b) in
@@CI's analysis) is NOT being taken — coverage matters
more than the four extra job-step lines.

### Local validation (optional, recommended)

@@Alex 2026-05-21 surfaced new local-Linux capability via
lima-vm + sdme containers. See memory
`reference-local-linux-via-sdme.md` for invocation shape.
Local validation lap:

1. `limactl shell default sudo sdme fs ls` to see
   available distros.
2. `limactl shell default sudo sdme chan-build-ubuntu -r
   ubuntu` to create a container.
3. `limactl shell default sudo sdme start chan-build-ubuntu`.
4. `limactl shell default sudo sdme cp ` the chan workspace
   into the container, OR `git clone` from inside.
5. `limactl shell default sudo sdme exec chan-build-ubuntu
   /bin/sh -c 'apt update && apt install -y libwebkit2gtk-4.1-dev
   libayatana-appindicator3-dev librsvg2-dev libsoup-3.0-dev
   patchelf libglib2.0-dev && cd /path/to/chan && cargo clippy
   --all-targets'`.
6. If `sdme exec` chokes on pty, use the systemd-run fallback
   from the memory.

**Architecture caveat (load-bearing)**: local lima
containers are **aarch64**; GH Actions `ubuntu-latest` is
**x86_64**. Local pass means "apt packages exist + clippy
compiles on aarch64 Linux". x86_64-specific issues (none
expected for an apt + clippy fix; flagged for completeness)
still need CI on the real `ubuntu-latest` runner. Don't
treat local-green as canonical-green for the gate.

If local validation is skipped, that's fine — the
`workflow_dispatch` smoke against ci.yml is the canonical
empirical gate. Local-aarch64 just shortens the iteration
loop if the first CI rerun reds for an unrelated reason.

#### Why this is a `systacean-N` finding

`systacean-6` made the BGE-small model embed opt-in
(default builds drop the ~140 MB bundle); `systacean-7`
followed with the runtime resolver. The chan-drive
tests above were written assuming the model is present
at runtime (likely true on dev workstations because
contributors run `chan index download-model` once and
the cache persists). CI runners are fresh, no cache,
and ci.yml doesn't fetch the model.

This was hidden behind the GTK gap; ci-12's fix
unmasks it. Three reasonable fix shapes for the
follow-up systacean task:

* **(a)** Mark the model-dependent tests `#[ignore]` or
  feature-gate them on `--features embed-model`.
  Smallest delta; matches the post-systacean-6 reality
  that "embedding model is optional".
* **(b)** Make the affected tests bring their own tiny
  test-model fixture (vector zeros / deterministic
  shape) that doesn't require BGE-small. More effort;
  better long-term coverage.
* **(c)** Pre-fetch the model in ci.yml's test job
  (mirror `release.yml`'s release matrix fetch step).
  Adds ~30-60s + ~140 MB downloaded per ubuntu-latest
  CI run; expensive given default builds don't embed.
