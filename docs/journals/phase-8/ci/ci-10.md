### Item 1 — auto-fetch Apple notary log on failure

New step inserted between "Verify signature + stapled
notarization" and "Upload notarization diagnostics":

* **Step name**: `Fetch Apple notary log`
* **Gate**: `matrix.os == 'macos-latest' && failure()`
* **Logic**:
  1. `set +e` — best-effort; never propagate failures
     from this diagnostic step so the artifact upload
     that follows still captures the rest of the
     diagnostic surface.
  2. Parse submission id from
     `/tmp/chan-desktop-build.log` via
     `grep -oE 'id: [a-f0-9-]{36}' | tail -1 |
     awk '{print $2}'`. `tail -1` picks the most
     recent submit in case multiple invocations ran;
     UUID regex is the notarytool output format.
  3. If no submission id found → `::warning::` +
     `exit 0` (still lets upload-diagnostics fire with
     the build log).
  4. `xcrun notarytool log <id>` with the three
     `APPLE_*` env vars + final positional path arg
     `/tmp/notary-log.json` (notarytool's documented
     "write JSON to this file" shape).
  5. `cat` the JSON inline for log-viewer visibility
     in addition to the artifact capture.
* **Artifact integration**: added `/tmp/notary-log.json`
  to the existing "Upload notarization diagnostics"
  step's `path` list. Same artifact name
  (`notarization-diagnostics-<arch>`), same 14-day
  retention. Future notarization failures will land the
  JSON automatically — saves the human round-trip to
  @@Alex's local Keychain that dryrun.3 needed.

WHY-comment block above the step documents the design
intent (load-bearing artifact for triage; removes
manual notarytool query round-trip).

### Item 2 — drop `_x64` suffix from DMG filename

**Architectural finding**: tauri-bundler 2.x does NOT
expose a filename template knob via `tauri.conf.json`.
The DMG name is hardcoded as
`<ProductName>_<version>_x64.dmg` in
`tauri-bundler/src/bundle/macos/dmg.rs` regardless of
target arch (`aarch64-apple-darwin` still produces
`_x64`). No `bundle.macOS.dmg.fileName` or equivalent
in the Tauri 2 config schema.

**Routing**: did NOT touch `tauri.conf.json`. The
authorization covered both surfaces but the right
surface for this fix is the workflow itself
(post-build rename). Configuration would have been
preferred — bundler doesn't offer it.

**Implementation**: new step inserted after the verify
step, before the failure-only steps:

* **Step name**: `Drop _x64 suffix from DMG filename`
* **Gate**: `matrix.os == 'macos-latest'` (runs on
  success, before any failure paths).
* **Logic**: list the .dmg, run `sed
  's/_x64\.dmg$/.dmg/'` over the path, `mv` if the
  pattern matched. Echo "skipping" if no `_x64`
  present (forward-compat: if tauri-bundler ever fixes
  the default, the step becomes a no-op rather than a
  hard failure).

WHY-comment block notes the rationale (universal2 on
the horizon; suffix is misleading on aarch64 today and
on universal2 tomorrow) + flags revisit-if-bundler-
gains-a-knob.

Per the architect lean in the task body, picked **drop
entirely** over `_aarch64` rename. Single artifact
filename across all future architectures matches the
universal-DMG end state.
