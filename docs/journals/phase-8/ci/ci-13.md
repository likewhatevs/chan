## Background

The gate-unblocker cascade since `ci-12` revealed a deep
Windows-specific debt surface across multiple lanes:

* `systacean-17` boxed `toml::Error` for `result_large_err`
  on Windows. ✓ Useful cross-platform improvement;
  preserved.
* `fullstack-b-24` cleared 14 Windows dead_code lints
  across `chan-server` + `chan-desktop` IPC code. ✓
  Useful cross-platform `#[cfg(unix)]` discipline;
  preserved.
* `systacean-18` + 4 follow-ups gated 28 BGE-dependent
  tests via `#[ignore]` (cross-platform; Ubuntu impact
  only — BGE model never present on CI runners). ✓
  Preserved.
* `systacean-20` gated 3 chan-drive lock-contract tests
  via `#[cfg(unix)]` (Windows lock primitive doesn't
  surface `DriveLocked`). ✓ Preserved.
* `systacean-20` smoke #2 + #3 fixups gated the watcher
  test + supporting helpers/imports — iterative
  per-symbol cascade still unmasking dead_code on
  Windows (smoke #4 surfaced 3 MORE: `Collector` +
  `wait_for` etc.).
* Plus NEW Windows test failures surfaced at the next
  layer: `terminal_sessions::tests` (7 failures in
  `chan-server` lib including
  `spawn_uses_configured_default_term`,
  `api_restart_terminal_respawns_same_session_command`,
  `conditional_pty_programs_validate_real_terminal`,
  `dispatch_agent_event_uses_chord_in_agent_mode`,
  etc.).

The cumulative iteration cost on Windows-specific
issues is no longer worth it given @@Alex's framing.
Drop Windows from the per-PR gate; revisit when
Windows becomes a real-user surface.

## Decision: drop Windows from `ci.yml` per-PR gate

Two scopes:

### `ci.yml` (per-PR gate)

Remove the `clippy + test (windows-latest)` matrix
entry. The matrix shrinks to `ubuntu-latest` +
`macos-latest`. All other steps unchanged (rustfmt,
build no-default-features, web check + test + build).

### `release.yml` (release artifacts)

Audit current matrix. Per `ci-7` + the Linux-binaries
bug-list entry, `release.yml` matrix likely has a
Windows target (chan CLI as .msi or .exe or similar).

* If Windows is in the `release.yml` matrix: drop it.
  Windows chan CLI binaries don't ship on the next
  tag (or any subsequent tag) until @@Alex revisits.
* If Windows is NOT in `release.yml` matrix: no-op.

`release-desktop.yml` is macOS-only DMG (+ Linux .deb
/ .AppImage); Windows chan-desktop isn't shipping yet
either. Unaffected.
