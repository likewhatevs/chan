## Decision: drop Windows from `ci.yml` per-PR gate

Two scopes:

### `ci.yml` (per-PR gate)

Remove the `clippy + test (windows-latest)` matrix
entry. The matrix shrinks to `ubuntu-latest` +
`macos-latest`. All other steps unchanged (rustfmt,
build no-default-features, web check + test + build).

### `release.yml` (release artifacts)

Audit current matrix. Per `ci-7` + the Linux-binaries
bug-list entry, `release.yml` matrix likely has a
Windows target (chan CLI as .msi or .exe or similar).

* If Windows is in the `release.yml` matrix: drop it.
  Windows chan CLI binaries don't ship on the next
  tag (or any subsequent tag) until @@Alex revisits.
* If Windows is NOT in `release.yml` matrix: no-op.

`release-desktop.yml` is macOS-only DMG (+ Linux .deb
/ .AppImage); Windows chan-desktop isn't shipping yet
either. Unaffected.
