# bob-1: Bump chan-core path deps to 0.11.0 + migration pass

Owner: Bob. Depends on: nothing. Unblocks: bob-2, martin-1.

## Goal

Bring ../chan onto chan-core 0.11.0 cleanly: regenerate `Cargo.lock`
and fix any compile breaks the new chan-llm CLI-oriented API
surfaces. End state is a green `cargo build`, `cargo test`,
`cargo fmt --check`, and `cargo clippy --all-targets -- -D warnings`.

Note: `Cargo.toml`'s four `[workspace.dependencies]` pins were
already bumped from `=0.10.0` to `=0.11.0` out of band before this
task was dispatched. Step 1 below is therefore informational; verify
the file matches what's described, but do not re-edit it.

## Background

chan-core 0.11.0 is already published in
`~/dev/github.com/chan-writer/chan-core` (see its workspace
`version = "0.11.0"`). It introduces `AgentStatus`, `AgentActivity`,
`UserRequest`, plus the matching `SessionListener` callbacks (defaults
provided), and may have renamed/removed older llm backend/status
concepts that ../chan still references.

Plan section "Prerequisite" warns this is a migration pass, not just
a version edit. Expect breakage in `crates/chan-server/src/routes/llm.rs`
and any other place that imports types from `chan_llm`.

## Files to touch

- `Cargo.toml` (workspace dependency table, four `=0.10.0` pins).
- `Cargo.lock` (regenerated by cargo).
- Whatever consumer code the compiler complains about. Most likely:
  - `crates/chan-server/src/routes/llm.rs`
  - `crates/chan-server/src/bus.rs`
  - possibly `crates/chan/src/main.rs` if it touches llm types.

Do not edit anything outside the consumer crates; do not vendor
chan-core changes back into this repo.

   matching `chan-drive`, `chan-llm`, `chan-tunnel-client`,
   `chan-tunnel-proto` already pin `version = "=0.11.0"`. Confirm,
